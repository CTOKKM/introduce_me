Artificial Intelligence

알고리즘이란?
>>> 특정한 일이나 문제를 해결하기 위하여 수행하는 해결 과정 혹은 방법을 의미한다. 특정한 일을 수행하며 문제를 해결해 간다는 점에서 프로그램과 동일하나, 알고리즘은 문제를 풀어가는 과정이 유한한 명령어들의 집합으로 해결하고, 프로그램은 유한성이 만족되지 않아도 된다는 점에서 구별된다. 즉, 알고리즘은 특정한 일이나 문제를 해결하기 위하여 만든 명령어들의 유한 집합(Finite Set)이라고 할 수 있다.

알고리즘 조건

  • 입출력(I/O) : 입력에 따른 결과를 출력한다.
  • 명확성(Clarity) : 각 명령어는 명확한 의미를 가진다.
  • 유한성(Endlichkeit) : 알고리즘은 반드시 수행 후 종료된다.
  • 유효성(Availability) : 실행되는 모든 명령어는 오류 없이 실행할 수 있어야 한다.
  • 효율성(Efficiency) : 입력값에 따라 효율적으로 수행되어야 한다.

알고리즘 선택 기준

  • 정확성(Accuracy) : 입력에 따른 결과가 정확한지의 여부이다.
  • 효율성(Efficiency) : 시간복잡도 및 공간복잡도가 효율적인지의 여부이다.
  • 적정성(Appropriateness) : 하드웨어 혹은 소프트웨어에서 해당 알고리즘이 적정한지의 여부이다.

알고리즘 유형

  • 탐욕법(Greedy Method) : 해당 단계에서 최선을 선택하여 결국 최적의 해를 구하는 기법으로, 크루스칼, 다익스트라, 허프만 코딩 등이 있다.
  • 분할과 정복(Divde and Conquer) : 작은 부분으로 분할하여 해를 구하고, 병합해가는 기법으로, 퀵 정렬, 병합 정렬 등이 있다.
  • 동적 계획법(Dynamic Programming) : 큰 하나의 문제 해결이 어려운 경우 작게 나누어 풀어 해당 문제의 값을 저장하여 재사용하는 기법으로, 피보나치 수가 대표적이다.
  • 백 트래킹(Back Tracking) : 여러 후보 해 중 특정 조건을 충족하는 모든 해를 구하기 위해 이전으로 돌아가며 해를 찾는 기법으로, N-Queen 알고리즘이 대표적이다.

알고리즘 복잡도

  • 시간 복잡도(Time Complexity) : 입력 값에 대하여 얼마만큼의 연산을 수행하는지 계산하여 알고리즘의 수행시간을 평가하는 척도이다.
  • 공간 복잡도(Space Complexity) : 알고리즘을 수행할 때 메모리가 얼마만큼 필요한지에 대하여 평가하는 척도이다.
